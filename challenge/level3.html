📡 Modbus TCP Frame Structure

MBAP Header (7 bytes) precedes every frame:
TxID[2] | Proto[2] | Len[2] | UnitID[1]

Protocol ID is always 0x0000 for Modbus.
Length counts bytes from Unit ID to end of PDU.
Unit ID maps to the slave device (0–247).

🔢 Function Codes

  • FC01 Read Coils — bit outputs
  • FC02 Read Discrete Inputs — bit inputs
  • FC03 Read Holding Registers — R/W words
  • FC04 Read Input Registers — read-only words
  • FC05 Write Single Coil
  • FC06 Write Single Register
  • FC16 Write Multiple Registers

🗺️ Register Address Spaces

  • 0x Coils — bit-addressable outputs (R/W)
  • 1x Discrete Inputs — bit-addressable inputs (RO)
  • 3x Input Registers — 16-bit read-only words
  • 4x Holding Registers — 16-bit R/W words
  • Byte order: Big-Endian (MSB first)
  • Max 125 registers per FC03/FC04 request

⚙️ Industrial Context

Modbus TCP runs over standard Ethernet on port 502. It is the most widely deployed industrial protocol — found in VFDs, PLCs, sensors, and HMIs from every major manufacturer. Understanding how to read a Modbus frame from a packet capture (Wireshark) is a core skill for field engineers.
