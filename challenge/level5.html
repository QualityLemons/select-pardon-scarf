⏱ TON — Timer On-Delay

TON starts counting when the IN (enable) rung goes TRUE. While counting, T0.TT = 1 (timer timing). When elapsed time T0.ET reaches the preset T0.PT, the done bit T0.DN fires. If IN goes FALSE at any point, the timer resets to zero immediately — this is why E-Stop cancels the cycle.

🔄 The Latch-Then-Time Pattern

The sensor pulse (I0) is momentary — the box only crosses the beam briefly. Rung 1 uses a seal-in contact on M0 to hold the timer enabled after I0 drops out. The /T0.DN NC contact in Rung 1 breaks the latch when the timer finishes, resetting the circuit for the next box.

🛑 E-Stop Resets the Timer

Because I1 (E-Stop NC) is in Rung 2, pressing E-Stop de-energises the TON's IN pin. TON immediately resets T0.ET → 0 and clears T0.TT. The motor also stops because Q0 follows the timer's enable state. This prevents half-moved boxes from being abandoned in unknown positions.

🏭 Real-World Use

  • TON — delay before starting (lubrication, warm-up)
  • TOF — motor runs for N seconds after stop command
  • RTO — retentive: timer accumulates across power cycles
  • CTU/CTD — count boxes, parts, or revolutions
  • Timers prevent pile-ups, ensure dwell times, and gate sequences
