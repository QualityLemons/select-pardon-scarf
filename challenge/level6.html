⚙ State Machine Basics

A state machine allows only one step to be active at a time. Each step (M-bit) seals itself in and holds until its exit condition fires. The mutual exclusion is implicit: M1's exit condition (/I1) kills it before M2's entry (I1) turns on — ensuring no overlap across a single PLC scan.

🔄 Seal-In + Exit Pattern

Every state rung follows the same pattern: (entry OR seal) AND exit_condition AND safety. The entry fires once to latch in; the exit-condition NC contact is what drops the state when the transition is met. This is the foundation of all sequential batch control.

🏭 ISA-88 Batch Standard

  • Phase: one state step (Fill, Mix, Drain)
  • Operation: a sequence of phases
  • Procedure: multiple operations = a full recipe
  • Used in pharma, food, chemicals — any recipe-driven process
  • SCADA shows real-time phase state exactly like this simulation

📊 Why Not Timers for Fill/Drain?

Open-loop timers assume a fixed fill rate. Sensor-driven transitions (LS_HIGH, LS_LOW) are closed-loop — they handle variable flow rates, pump wear, and partially-full tanks automatically. Timers are used only where a sensor cannot measure the condition (mixing quality).
