01 / KEY TERMS

THE JARGON, DECODED

25 terms every PLC engineer knows — filtered by category or searched by name.

PLC
PROGRAMMABLE LOGIC CONTROLLER
HARDWARE

A ruggedised industrial computer designed to control machinery in real time. Unlike a PC, it runs the same program in a continuous loop (the scan cycle) and is built to survive vibration, temperature extremes, and electrical noise on a factory floor.

e.g. Siemens S7-1200 · Allen-Bradley CompactLogix · Mitsubishi FX5U
CPU MODULE
CENTRAL PROCESSING UNIT
HARDWARE

The "brain" of the PLC rack. It stores the user program, executes the scan cycle, and manages communication with I/O modules. Most modern CPUs also include built-in Ethernet ports and onboard I/O.

e.g. Siemens CPU 1214C has 14 digital inputs, 10 digital outputs, 2 analogue inputs built in
I/O MODULE
INPUT / OUTPUT
HARDWARE

Physical cards that connect the PLC to the real world. Input modules read sensors and switches. Output modules drive actuators, contactors, and lights. Available in digital (on/off) and analogue (0-10 V, 4-20 mA) variants.

e.g. DI 8×24VDC reads eight 24 V digital switches simultaneously
SENSOR
FIELD DETECTION DEVICE
HARDWARE

A device that converts a physical condition (position, pressure, temperature, level) into an electrical signal the PLC can read. Common types: proximity (detects metal without contact), photoelectric (light beam), limit switch (mechanical contact).

e.g. LS_HIGH sensor on tank tells PLC when liquid reaches maximum level
ACTUATOR
OUTPUT / FIELD DEVICE
HARDWARE

Any device the PLC drives as an output — motors, solenoid valves, contactors, warning lights, and alarms. The PLC's output module switches 24 V DC to energise these devices.

e.g. Q0 output energises a contactor, which closes to start the pump motor
RELAY
ELECTROMECHANICAL SWITCH
HARDWARE

A coil-driven switch that uses a small control signal to open or close a higher-power circuit. Widely used to isolate PLC outputs from high-voltage loads, and as the physical metaphor for ladder logic contacts and coils.

e.g. 24 V coil closes a set of contacts rated for 230 V AC motor power
HMI
HUMAN-MACHINE INTERFACE
HARDWARE

The touchscreen panel (or PC-based software) used by operators to monitor and control the machine. Displays live PLC data — tank levels, alarm status, motor speeds — and lets operators issue commands without touching the PLC directly.

e.g. Siemens KTP700 Basic 7" touchscreen showing tank fill percentage
24 V DC
INDUSTRY STANDARD CONTROL VOLTAGE
HARDWARE

The standard control voltage used in almost all modern industrial panels. The 24 V PSU converts 230 V AC mains into a stable, safe DC supply for PLCs, sensors, and I/O modules. Low enough to reduce shock risk; high enough to be noise-immune.

e.g. Phoenix Contact QUINT PSU: 230 V AC in → 24 V DC, 10 A out
LADDER LOGIC
IEC 61131-3 LANGUAGE: LD
LOGIC

The most common PLC programming language. Graphical and intentionally designed to look like relay wiring diagrams so electricians could transition to software in the 1960s. Each horizontal "rung" is a logical expression that controls an output coil.

e.g. ──[I0]──[/I1]──(Q0)── reads "if I0 is ON and I1 is OFF, energise Q0"
RUNG
ONE LINE OF LADDER LOGIC
LOGIC

A single horizontal line in a ladder diagram. The PLC evaluates each rung left-to-right during the scan cycle. If the logic resolves to TRUE (the "power" reaches the right rail), the output coil is energised. Multiple rungs can share inputs.

e.g. A 10-rung program might have 3 rungs for motor control, 4 for alarms, 3 for timers
CONTACT
NORMALLY OPEN / NORMALLY CLOSED
LOGIC

A contact reads a bit value. NO (Normally Open) — passes power when the bit is 1 (TRUE). NC (Normally Closed) — passes power when the bit is 0 (FALSE). NC contacts are used for E-Stop and fault conditions so a wire break fails safe.

e.g. [I0] = NO contact on Start button · [/I1] = NC contact on Stop/E-Stop
COIL
OUTPUT ELEMENT
LOGIC

The output at the end of a rung — shown as (Q0). When the rung logic is TRUE, the coil is energised (bit = 1), turning on the associated output. Named after the physical relay coil it replaced in early control panels.

e.g. (Q0) energised → output module delivers 24 V DC → contactor closes → motor starts
SCAN CYCLE
THE PLC'S MAIN LOOP
LOGIC

The repetitive process the PLC runs continuously: 1. Read Inputs → copy all field signals into memory. 2. Execute Program → evaluate all rungs using the snapshot. 3. Write Outputs → push results to field devices. Typically 1–50 ms per cycle.

e.g. PLeC simulates a 50 ms scan cycle — fast enough for most industrial processes
TAG
NAMED MEMORY ADDRESS
LOGIC

A human-readable name for a PLC memory address. Instead of writing raw addresses like I0.0, engineers use tags like START_PB or MOTOR_RUN. Makes programs far easier to read, debug, and maintain.

e.g. I0.0 → "START_PB" · Q0.0 → "MOTOR_FWD" · M0.0 → "LATCH_BIT"
LATCH / SEAL-IN
SELF-HOLDING CIRCUIT
LOGIC

A circuit pattern where the output's own contact is wired in parallel with the Start input. Once energised, Q0 holds itself ON even after the Start button is released. The only way to break it is the Stop/E-Stop opening the series NC contact.

e.g. ──[START]──┬──[/STOP]──(MOTOR)── with MOTOR's NO contact on the ┬ branch
TIMER
TON / TOF / TP INSTRUCTIONS
LOGIC

TON (On-Delay) — starts counting when input goes TRUE; output fires after the preset time. TOF (Off-Delay) — output stays on for a set time after input drops. Timers have EN (enable), TT (timing), DN (done), ET (elapsed) bits.

e.g. TON T0, PT=10s: after 10 s, T0.DN = 1 → next rung sees the done bit
COUNTER
CTU / CTD INSTRUCTIONS
LOGIC

CTU (Count Up) — increments on each rising edge of the CU input. When accumulated value (ACV) reaches the preset value (PV), the done bit fires. Used for counting products, pallets, pulses from encoders.

e.g. CTU C0, PV=100: counts 100 boxes on a conveyor, then triggers a divert gate
MODBUS
SERIAL / TCP PROTOCOL
COMMS

The oldest and most universal industrial protocol (1979). Modbus RTU runs over RS-485 serial. Modbus TCP wraps the same frames in Ethernet packets. Function codes read/write coils (bits) and holding registers (16-bit words).

e.g. FC03 Read Holding Registers: master sends request → slave returns 10 register values
PROFINET / EtherNet/IP
REAL-TIME ETHERNET PROTOCOLS
COMMS

High-speed Industrial Ethernet protocols. PROFINET is Siemens-standard; EtherNet/IP is Rockwell/ODVA standard. Both run at 100 Mbit/s or 1 Gbit/s over standard Cat5e/6 cable and support sub-millisecond cycle times for motion control.

e.g. PROFINET connects a Siemens PLC to a remote I/O rack 50 metres away via Cat6
SCADA
SUPERVISORY CONTROL & DATA ACQUISITION
COMMS

Software that connects to multiple PLCs, collects data, displays plant-wide overviews, logs historical trends, and generates alarms. The operator's window into the entire plant — from a single workstation or web browser.

e.g. Ignition SCADA, WinCC, InTouch — used in water treatment, power, food & beverage
RS-485
SERIAL BUS STANDARD
COMMS

A 2-wire (A/B) serial bus standard that allows up to 32 devices on a single cable run of up to 1200 metres. The backbone of Modbus RTU and many older fieldbus systems. Still extremely common in industrial installations worldwide.

e.g. One RS-485 cable daisy-chains a PLC to 16 inverter drives, each with a unique slave address
E-STOP
EMERGENCY STOP DEVICE
SAFETY

A red mushroom-head button that immediately de-energises all dangerous outputs when pressed. Wired as NC in series with the control circuit — pressing it breaks the circuit, dropping all outputs to safe state. Must be manually released before the machine can restart.

e.g. IEC 60947-5-5 specifies E-Stop requirements; PSSR 2000 requires them on UK installations
FAIL-SAFE
SAFE-STATE ON FAILURE
SAFETY

Design principle: if any component fails (wire breaks, sensor fails, power lost), the system moves to a safe state — typically with all dangerous outputs de-energised. Achieved by wiring safety devices as NC so a wire break = de-energised = safe.

e.g. LS_HIGH wired NC: broken wire → I1 reads 0 → PLC sees "tank full" → pump stops ✓
IEC 61131-3
PLC PROGRAMMING STANDARD
SAFETY

The international standard defining five PLC programming languages: Ladder Diagram (LD), Function Block Diagram (FBD), Structured Text (ST), Instruction List (IL), and Sequential Function Chart (SFC). Ensures programs are portable across compliant PLCs.

e.g. OpenPLC and Codesys both implement all five IEC 61131-3 languages
SIL
SAFETY INTEGRITY LEVEL
SAFETY

A rating (SIL 1–4) from IEC 62061 that quantifies how reliably a safety function reduces risk. Higher SIL = lower probability of dangerous failure per hour. Dual-channel E-Stop circuits with cross-monitoring typically achieve SIL 2 or SIL 3.

e.g. SIL 2 requires a probability of dangerous failure < 10⁻⁷ per hour of operation
PSSR 2000
PRESSURE SYSTEMS SAFETY REGULATIONS
SAFETY

UK legislation requiring all pressure systems (including pneumatic control systems) to have a written scheme of examination, adequate protection devices, and safe operating limits. Violations can result in prosecution. Referenced in Level 4 Safety Interlock challenge.

e.g. Pneumatic drill press controlled by a PLC must comply with PSSR 2000 inspection schedules
